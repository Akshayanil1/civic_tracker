# Domain-Specific Vocabulary Flashcards

Use these terms to elevate your technical English during university interviews or IELTS speaking exams.

1. **Multi-Tenant Architecture**
   - *Definition:* A software architecture where a single instance of the software serves multiple distinct user groups (tenants), utilizing isolated databases.
   - *Usage:* "I scaled the system using a multi-tenant architecture to support multiple municipal corporations simultaneously."

2. **Asynchronous Processing**
   - *Definition:* Executing operations independently of the main program flow, usually via background queues.
   - *Usage:* "I relied on asynchronous processing via Redis queues to handle heavy penalty calculations without blocking the main thread."

3. **Headless ERP**
   - *Definition:* An Enterprise Resource Planning backend that exposes its functionalities strictly through APIs, decoupling it from the frontend presentation layer.
   - *Usage:* "I decoupled the citizen portal by utilizing Frappe as a headless ERP."

4. **Service Level Agreement (SLA)**
   - *Definition:* A documented agreement defining the expected level of service and resolution timelines.
   - *Usage:* "The core innovation was algorithmically enforcing SLA deadlines to ensure contractor accountability."

5. **Omnichannel Ingestion**
   - *Definition:* The ability to receive data seamlessly across multiple different platforms (Web, Mobile, WhatsApp, IoT).
   - *Usage:* "I built an omnichannel ingestion pipeline to capture data from web forms, WhatsApp bots, and IoT sensors."

6. **Triage / Triaging**
   - *Definition:* The process of determining the priority of tasks or problems based on severity.
   - *Usage:* "I integrated an AI layer for automated triaging, dynamically prioritizing high-urgency crises."

7. **Personally Identifiable Information (PII)**
   - *Definition:* Any data that could potentially identify a specific individual.
   - *Usage:* "The Open Data API was strictly programmed to scrub all PII to maintain citizen anonymity."
